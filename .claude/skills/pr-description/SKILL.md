PR 설명 작성 시:

1. `git diff main...HEAD`를 실행하여 이 브랜치의 모든 변경사항 확인
2. 다음 형식에 따라 설명 작성:

## What (무엇을)
이 PR이 무엇을 하는지 한 문장으로 설명.

## Why (왜)
이 변경이 필요한 이유에 대한 간단한 배경

## Changes (변경사항)
- 구체적인 변경사항을 불릿 포인트로 나열
- 관련 변경사항은 묶어서 정리
- 삭제되거나 이름이 변경된 파일 언급
